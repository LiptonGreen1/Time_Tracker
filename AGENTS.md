# Time Tracker PWA — Claude.md

## 1. Цель проекта

Создать **мобильное PWA-приложение для учета времени** (iPhone → Safari → Add to Home Screen).

Суть:

* Пользователь фиксирует **100% своего времени (24/7)**
* 1 нажатие = смена деятельности
* Система строит **непрерывную временную линию жизни**

Это НЕ просто таймер.
Это **система учета ресурса времени** (аналог финансового учета, но для времени).

---

## 2. Базовые принципы (КРИТИЧНО)

1. **Нулевая фрикция**

   * 1 тап = смена активности
   * Без подтверждений
   * Без лишних окон

2. **Непрерывность**

   * Таймлайн должен быть **непрерывным**
   * Без дыр (если не задано явно)

3. **Целостность данных > интерфейс**

   * Нельзя ломать записи
   * Нельзя терять историю

4. **Append-only мышление**

   * Не перезаписывать, а трансформировать
   * Разрезать записи вместо удаления

5. **Mobile-first**

   * Интерфейс под iPhone
   * Крупные кнопки

## 4. Модель данных

```js
state = {
  groups: [
    {id, name}
  ],

  activities: [
    {id, name, group}
  ],

  records: [
    {
      id,
      activity,
      start, // ISO строка
      end    // ISO или null
    }
  ],

  active: record_id | null
}
```

Правила:

* В каждый момент времени **только одна активная запись**
* Активная запись имеет `end = null`
* Все даты — в ISO формате

---

## 5. Основная логика

### 5.1 Запуск активности

При нажатии:

1. Завершить текущую активность (`end = now`)
2. Создать новую запись:

   * start = now
   * end = null
3. Сделать её активной

---

### 5.2 Целостность таймлайна

Все записи должны:

* Не пересекаться
* Быть отсортированы по времени
* Отражать реальную последовательность

---

## 6. КРИТИЧНАЯ ФУНКЦИЯ — вставка задним числом

Пользователь может вставить активность в прошлом.

Пример:

До:

```
16:00 — 18:00 Читал
```

Вставка:

```
17:00 — 17:30 Гулял
```

После:

```
16:00 — 17:00 Читал
17:00 — 17:30 Гулял
17:30 — 18:00 Читал
```

---

### Алгоритм (ОБЯЗАТЕЛЕН)

Для интервала [start, end]:

Для каждой записи:

Если есть пересечение:

* если record.start < start → сохранить левую часть
* если record.end > end → сохранить правую часть
* исходную запись удалить

После:

* добавить новую запись
* отсортировать
* объединить соседние одинаковые активности
